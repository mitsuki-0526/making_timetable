# 時間割作成ツール（開発中）

## 1. プロジェクト概要
学校の時間割作成業務を効率化するハイブリッド型ツール。
詳細な定義については、[PROJECT_DEFINITION.md](./docs/PROJECT_DEFINITION.md) を参照してください。

## 2. システム構成
- **フェーズ1 (AI)**: ローカルAI (Gemma) による条件抽出。
- **フェーズ2 (Program)**: JSアルゴリズムによる自動初期配置。
- **フェーズ3 (Human)**: 直感的なドラッグ＆ドロップUIによる最終調整。

## 3. 技術要件
- **実行環境:** Windows OS
- **ソースコード管理:** GitHub
- **フロントエンド:** React + Vite + Zustand
- **AI部:** Local LLM (Google AI Studio を介した検証から、ローカルGemmaへの移行を計画)

## 3. コア機能要件

### 3.1. 基本UIとスケジュール管理
* **時間割グリッド:** 月曜日〜金曜日、各日1限〜6限（標準）のスケジュール枠。
* **学年・クラス設定:** * 各学年ごとのクラス数（例: 1年生は4クラス、2年生は3クラスなど）を柔軟に設定・変更できる機能。
* **コマの配置:** 各クラスの曜日・時限のコマに対して、「教科」と「担当教員」を割り当てる機能。

### 3.2. 動的プルダウン制御（誤配置の防止機能）
時間割のコマ（セル）をクリックして教員を割り当てる際、以下の条件をリアルタイムに計算し、**配置可能な教員のみをプルダウンに表示**する。
* **担当学年フィルタ:** 編集中のコマの学年を担当している教員のみを表示。
* **要望（配置不可時間）フィルタ:** 教員があらかじめ「配置不可（出張、会議など）」として設定した曜日・時限では、その教員を候補から除外。
* **重複配置フィルタ:** 既に同日・同時限の他クラスに配置されている教員は候補から除外。

### 3.3. バリデーション機能（リアルタイム警告）
* **授業時数チェック:** 学年・クラス・教科ごとに規定された「週当たりの授業時数」に対して、現在の配置時数を自動カウントし、過不足をUI上で可視化する。
* **同日・同教科の重複チェック:** 同じクラスの同じ日に、同一教科が複数回配置された場合に警告表示を行う（意図的な2時間連続授業などの例外対応も考慮する）。

## 4. データ構造（設計のベースライン）

開発にあたり、以下のデータ構造（JSON相当）を状態管理として保持・永続化すること。

**1. 教員データ (`Teacher`)**
- `id`: 教員ID
- `name`: 教員名
- `subjects`: 担当教科リスト
- `target_grades`: 担当学年リスト
- `unavailable_times`: 配置不可な日時（曜日と時限の配列）

**2. クラス構成データ (`SchoolStructure`)**
- `grades`: 各学年のクラス数

**3. 時間割データ (`TimetableEntry`)**
- `day_of_week`: 曜日
- `period`: 時限
- `grade`: 学年
- `class_name`: クラス名
- `subject`: 教科
- `teacher_id`: 担当教員ID

## 6. 開発・デプロイ手順

### 開発サーバーの起動

```bash
pnpm run dev
```

ブラウザで `http://localhost:5173/making_timetable/` を開いてください。

### GitHub Pages へのデプロイ

> ⚠️ `git push` だけでは GitHub Pages は更新されません。必ず以下の手順を実行してください。

```bash
# 1. ソースコードを main ブランチへプッシュ（通常の開発フロー）
git add .
git commit -m "コミットメッセージ"
git push origin main

# 2. GitHub Pages 用にビルドしてデプロイ（gh-pages ブランチへ）
pnpm run deploy
```

`pnpm run deploy` は内部で以下を実行します：
1. `vite build` — `/making_timetable/` をベースパスとして本番ビルドを生成（`dist/` フォルダ）
2. `gh-pages -d dist` — `dist/` の内容を `gh-pages` ブランチにプッシュ

### GitHub Pages の初期設定（初回のみ）

リポジトリの Settings → Pages で以下を設定してください：

- **Source**: `Deploy from a branch`
- **Branch**: `gh-pages` / `/ (root)`

> `main` ブランチを指定すると未ビルドのソースファイルが公開されてしまい、404エラーが発生します。必ず `gh-pages` ブランチを選択してください。
