# AGENTS.md

このリポジトリの開発・運用における基本ルールをまとめたエージェント向けドキュメントです。
`AGENTS.md` は、共通のワークフローとツール選択をチーム全体で共有するためのガイドです。

## 1. ドキュメント関連

- このリポジトリには `docs/` フォルダがあります。
  - `docs/` 以下のドキュメントは、設計方針・仕様・プロジェクト定義の一次情報です。
  - 仕様変更や機能追加を行うときは、`docs/` の内容を適宜更新してください。
    - 例: UI変更、動作ロジック追加、ビルド/パッケージング方針変更など。
  - 主要なドキュメントとしては、`docs/PROJECT_DEFINITION.md` が存在します。ここでプロジェクト全体の目的と構成を確認できます。

## 2. パッケージ管理

- このリポジトリでは `npm` / `npx` ではなく、`pnpm` を使用します。
- 依存関係のインストールには `pnpm install` を使ってください。
- スクリプト実行も `pnpm run <script>` を使います。
  - 例: `pnpm run dev`, `pnpm run build`, `pnpm run format`, `pnpm run lint`
- 可能なかぎりプロジェクトの `package.json` に定義済みのスクリプトを利用してください。

## 3. フォーマッタ / リンター

- このリポジトリでは `biome` をコード整形と静的チェックの標準ツールとして採用しています。
- ルートの `package.json` には以下のスクリプトが定義されています。
  - `pnpm run format` → `biome format`
  - `pnpm run lint` → `biome check`
- `biome` の設定は `biome.jsonc` で管理されています。
- 変更前後でコード整形や静的チェックを通すことで、レビューがスムーズになります。

## 4. 作業完了時の確認

- コード変更を完了する前に、以下のコマンドを実行してコード品質を保ってください。
  - `pnpm run format` → コード整形
  - `pnpm run lint` → 静的コードチェック
  - `pnpm run build` → ビルドエラーの確認

## 5. 作業上の注意点

- `dist/` や `node_modules/` は生成物/依存先のため、直接編集しないでください。
- ルート以外に `desktop/` というディレクトリがあります。
  - `desktop/` はデスクトップ版 Electron + Python バックエンドに関する別軸の実装です。
- UIや機能の仕様変更を行う際は、該当する `docs/` の内容も見直してください。
  - 可能であれば、ドキュメント修正と実装変更をセットで行うと内容の齟齬を防げます。
