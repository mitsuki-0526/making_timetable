# 時間割作成ツール — 要件定義書 (PRD)

バージョン: 2.0  
最終更新: 2026-04-09  
ステータス: ドラフト

---

## 1. プロジェクト概要

### 1.1 目的
中学校における時間割作成業務を効率化するWebアプリケーション。
教員の配置制約・規定時数・特別支援学級の連動など、複雑な条件をクリアした時間割を作成できる。

### 1.2 スコープ
- **対象**: Web版（GitHub Pages PWA）のみ
- **対象外**: Electron版、Python バックエンド、OR-Tools ソルバー（将来スコープ）
- **対象ブラウザ**: Chrome / Edge（最新2バージョン）
- **対象デバイス**: デスクトップPC（学校現場は Windows + Chrome/Edge が大半）

### 1.3 技術スタック

| 役割 | 技術 |
|------|------|
| 言語 | TypeScript |
| UI | React 18 + Vite 5 |
| 状態管理 | Zustand 5 |
| アイコン | lucide-react |
| テスト | Vitest + React Testing Library |
| デプロイ | GitHub Pages (gh-pages) |
| AI | Gemini API (gemini-2.0-flash-lite) |

---

## 2. ユーザーと利用シーン

### 2.1 主要ユーザー
- **教務主任**: 時間割全体の作成・調整を担当。1人で全校分を作成する
- **管理職（教頭・校長）**: 完成した時間割の確認・承認

### 2.2 利用フロー
1. マスタデータ登録（教員・クラス・教科・施設）
2. 制約条件設定（固定コマ・教員制約・教科配置制約）
3. ソルバーによる自動配置（80-95%完成）
4. 手動での微調整（バリデーション警告を見ながら）
5. PDF出力・印刷

## 3. データモデル

### 3.1 教員データ (`Teacher`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | 教員ID（UUID） |
| `name` | string | 教員名 |
| `subjects` | string[] | 担当可能教科リスト |
| `target_grades` | number[] | 担当学年リスト |
| `unavailable_times` | `{day: Day, period: number}[]` | 配置不可な曜日・時限 |
| `employment_type` | `"full_time" \| "part_time" \| "temporary" \| "substitute"` | 勤務形態（常勤/非常勤/講師/期限付き） |
| `available_days` | `Day[] \| null` | 出勤可能曜日（非常勤・講師用。nullなら全曜日） |
| `min_hours` | `number \| null` | 最低保証週時数（非常勤用。nullなら制約なし） |
| `max_hours` | `number \| null` | 最大週時数（nullなら制限なし） |
| `contract_end_date` | `string \| null` | 契約終了日（期限付き用。ISO 8601。nullなら無期限） |
| `is_homeroom` | `boolean` | 担任かどうか |
| `homeroom_class` | `{grade: number, class_name: string} \| null` | 担任クラス |

### 3.2 教員グループ (`TeacherGroup`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | グループID |
| `name` | string | グループ名（例: 「1年道徳」） |
| `teacher_ids` | string[] | 所属教員IDリスト |
| `subjects` | string[] | 担当教科リスト |
| `target_grades` | number[] | 対象学年リスト |

> 道徳・総合的な学習の時間など、複数教員で担当する教科用。
> グループに属する教員は個別の配置不可時間を持ち、グループとして配置される場合はグループ内の全教員がその時間に空いている必要がある。

### 3.3 学校構造 (`SchoolStructure`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `grades` | `GradeConfig[]` | 学年ごとの設定 |

#### GradeConfig

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `grade` | number | 学年（1, 2, 3） |
| `classes` | `ClassInfo[]` | クラス一覧 |
| `required_hours` | `Record<string, number>` | 教科別の週あたり規定時数 |

#### ClassInfo

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `name` | string | クラス名（例: 「1組」「特支A」） |
| `is_special_needs` | boolean | 特別支援学級かどうか |

### 3.4 時間割エントリ (`TimetableEntry`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `day_of_week` | Day | 曜日（'月'\|'火'\|'水'\|'木'\|'金'） |
| `period` | number | 時限（1〜6） |
| `grade` | number | 学年 |
| `class_name` | string | クラス名 |
| `subject` | `string \| null` | A週（通常）の教科 |
| `teacher_id` | `string \| null` | A週（通常）の担当教員ID |
| `alt_subject` | `string \| null` | B週の教科（隔週授業時） |
| `alt_teacher_id` | `string \| null` | B週の担当教員ID（隔週授業時） |
| `teacher_group_id` | `string \| null` | 教員グループID |
| `cell_group_id` | `string \| null` | 合同コマグループID |

#### セルキーのフォーマット
```
"grade|class_name|day_of_week|period"
例: "1|1組|月|1"
```

### 3.5 合同クラス (`ClassGroup`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | グループID |
| `grade` | number | 学年 |
| `class_names` | string[] | 合同対象のクラス名リスト |
| `split_subjects` | string[] | 分割教科リスト（合同クラスで別々の教員が教える教科） |

### 3.6 施設 (`Facility`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | 施設ID |
| `name` | string | 施設名（例: 「体育館」「音楽室」） |
| `capacity` | number | 同時使用可能クラス数（デフォルト: 1） |

### 3.7 設定 (`Settings`)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `mapping_rules` | `MappingRule[]` | 特支連動マッピングルール |
| `lunch_after_period` | number | 昼休み開始時限（この時限の後が昼休み。デフォルト: 4） |

#### MappingRule

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `grade` | number | 対象学年 |
| `from_subject` | string | 通常学級の教科 |
| `to_subject` | string | 特支学級に設定する教科 |

## 4. 機能要件

### 4.1 時間割グリッド

#### 4.1.1 クラス別ビュー
- 行 = クラス（各学年の通常クラス＋特支クラス）、列 = 曜日 × 時限
- 特支クラスは各学年の通常クラスの直下に表示
- 曜日ごとに区切り線を表示
- セルクリックで CellDropdown（教科・教員選択）を開く

#### 4.1.2 先生別ビュー
- 行 = 教員、列 = 週計 + 月〜金 × 1〜6時限
- 週コマ数を自動集計して表示
- 役割別の色分け:
  - 青: 主担当
  - 紫: B週担当
  - 緑: 教員グループ経由
  - 黄: 合同グループ or 特支クラス
- 合同グループは1スロットで1カウント（クラス数ではない）

#### 4.1.3 タブ切替
- 「クラス別」「先生別」をタブで切替

### 4.2 セル編集（動的プルダウン制御）

コマ（セル）に教科を配置する際、以下の条件でリアルタイムにフィルタリングする:

#### 4.2.1 教科選択
- 対象クラスの学年に設定されている教科のみ表示

#### 4.2.2 教員自動割り当て
教科を選択すると、以下の条件を満たす教員を**自動的に割り当てる**:
1. その教科を担当できる（`subjects` に含まれる）
2. その学年を担当している（`target_grades` に含まれる）
3. 配置不可時間に該当しない（`unavailable_times` に含まれない）
4. 非常勤・講師の場合、出勤可能曜日である（`available_days` に含まれる）
5. 同日・同時限に他クラスで配置済みでない
6. 教員グループ経由の割当ではない（グループ配置は別ルート）

**複数候補がいる場合の優先順位**: その週のコマ数が少ない教員を優先（ロードバランス）

#### 4.2.3 教員不足時の仮置き
担当可能な教員が全員配置済みの場合:
- 教科のみ仮置き（`teacher_id = null`）
- セルに赤字で「空きなし」警告を表示

#### 4.2.4 右クリックメニュー
- 教員変更（候補リストから手動選択）
- 隔週設定（B週の教科・教員を設定）
- グループ化/グループ解除

### 4.3 特別支援学級の自動連動

#### 4.3.1 基本動作（単方向）
- 通常学級にコマを配置すると、`settings.mapping_rules` に基づいて同学年の特支クラスへ自動的に対応教科をセット
- 連動方向: **通常学級 → 特支学級の単方向のみ**
- ルール未設定の教科: 連動しない（旧版のフォールバックコピーは廃止。明示的なルール設定を必須とする）

#### 4.3.2 手動上書き
- 特支クラスのコマを手動で直接編集可能
- 手動で上書きしたコマには「手動」フラグが付く
- 「手動」フラグ付きのコマは、通常学級の変更による自動連動の対象外（ロック状態）
- ロック解除操作で再び自動連動の対象に戻せる

#### 4.3.3 設計方針
- 連動ロジックはストラテジーパターンで実装し、将来的な双方向連動への拡張を容易にする

### 4.4 合同コマ（セルグループ化）

#### 4.4.1 操作
- Ctrl+クリックで複数セルを選択（青枠で強調）
- 選択中はバナー表示:「N セル選択中 — 右クリック → グループ化 / Esc で選択解除」
- 右クリック →「N セルをグループ化」でグループ作成
- グループ化解除も右クリックメニューから

#### 4.4.2 表示
- グループ内のセルは左枠に同色の太線（グループごとに色をローテーション）
- 「合同」バッジを表示

#### 4.4.3 動作
- グループ内のセルは教科変更時も `cell_group_id` を保持
- 先生別ビュー・PDF出力では合同グループの全クラスを1スロットにまとめて表示
- 合同グループは1スロットで1コマとしてカウント

### 4.5 隔週授業（A週/B週）

#### 4.5.1 データ構造
- A週: `subject` + `teacher_id`
- B週: `alt_subject` + `alt_teacher_id`

#### 4.5.2 操作
- A週に教科をセット後、右クリック or 詳細設定からB週の教科・教員を登録
- B週の教員も自動割り当てルール（4.2.2）に従う

#### 4.5.3 表示
- セル内にA/Bバッジ付きで両方の教科を表示
- PDF出力でも同一セル内に `[A]` `[B]` バッジ付きで表示

#### 4.5.4 定義
- A週/B週の切替は学校の独自運用に委ねる（アプリ側で切替日を管理しない）
- 隔週ペアの設定は制約設定モーダルから行う

### 4.6 抱き合わせ教科

#### 4.6.1 定義
- 2つのクラス間で、同じ曜日・同じ時限に特定の教科ペアを配置するルール
- 例: A組に「技術」を置いたら、同日同時限にB組に「家庭科」を自動配置

#### 4.6.2 動作
- **双方向**: A組に技術を置くとB組に家庭科が入る。B組に家庭科を置くとA組に技術が入る
- 対象は同一学年内のクラス間のみ

### 4.7 マスタ設定

タブ構成のモーダルで管理。各タブ:

| タブ | 設定内容 |
|------|---------|
| 教科設定 | 教科の追加・削除、学年別の規定時数設定、特支連動マッピングルール |
| クラス設定 | 学年・クラスの追加・削除、特支クラスの管理 |
| 教員設定 | 教員のCRUD、担当教科・学年・配置不可時間・勤務形態の編集 |
| 教員グループ | グループの作成・編集・削除。教員選択、教科・学年フィルタ |
| 合同クラス | 合同クラスの定義、分割教科の指定 |
| 抱き合わせ教科 | 教科ペアの登録・削除 |
| AI設定 | Gemini APIキーの登録・接続テスト・モデル選択 |

### 4.8 制約設定

タブ構成のモーダルで管理。各タブ:

| タブ | 設定内容 |
|------|---------|
| 固定コマ | 全校共通 / 学年単位 / クラス単位の固定コマ登録 |
| 時間帯設定 | 昼休みの位置（午前/午後の判定に使用） |
| 教員制約 | 1日最大コマ数、連続コマ数上限、週最大コマ数 |
| 教科配置制約 | 配置許可時限、連続授業日数上限、午後コマ上限、2時間連続許可 |
| 施設管理 | 施設マスタ、教科↔施設マッピング、同時使用上限 |
| 隔週ペア | 隔週授業のペア定義（A週教科/B週教科） |
| 連続配置 | 連続配置ペア（例: 数学→数学の2時間連続授業） |

#### 4.8.1 制約の硬さ（ハード/ソフト）

**システムでハード固定（必ず満たす）:**
- 教員の配置不可時間
- 教員の出勤可能曜日（非常勤）
- 施設の同時使用上限
- 固定コマ
- 同一教員の同日同時限の重複配置禁止

**ユーザーがハード/ソフトを選択可能:**
- 教員の1日最大コマ数
- 教員の連続コマ数上限
- 教員の週最大コマ数
- 教科の連続授業日数上限
- 教科の配置許可時限
- 教科の午後コマ上限

> ソフト制約 = ソルバーは可能な限り満たそうとするが、満たせない場合は違反してもよい。バリデーションパネルで警告表示する。

### 4.9 バリデーション（リアルタイム警告）

時間割の編集に連動して、以下の検証をリアルタイムで実行し、警告パネルに表示:

| 検証項目 | 説明 | 重要度 |
|---------|------|--------|
| 固定コマ違反 | 固定コマに設定された教科と異なる教科が配置されている | エラー |
| 教員重複 | 同一教員が同日同時限に複数クラスに配置されている | エラー |
| 施設競合 | 同日同時限に施設の同時使用上限を超えている | エラー |
| 規定時数不足 | クラス×教科の週コマ数が規定時数に達していない | 警告 |
| 規定時数超過 | クラス×教科の週コマ数が規定時数を超えている | 警告 |
| 教員日次上限 | 教員の1日のコマ数が上限を超えている | 警告/エラー（設定依存） |
| 教員連続上限 | 教員の連続コマ数が上限を超えている | 警告/エラー（設定依存） |
| 教員週次上限 | 教員の週コマ数が上限を超えている | 警告/エラー（設定依存） |
| 連続授業日数 | 同一教科が設定日数以上連続で配置されている | 警告/エラー（設定依存） |
| 教科配置時限 | 許可されていない時限に教科が配置されている | 警告/エラー（設定依存） |
| 午後コマ上限 | 教科の午後コマ数が上限を超えている | 警告/エラー（設定依存） |
| 非常勤出勤日 | 非常勤教員が出勤不可の曜日に配置されている | エラー |
| 未配置コマ | 教科未設定のコマが残っている | 情報 |

> 「警告/エラー（設定依存）」は、ユーザーがその制約をハード/ソフトどちらに設定したかで表示が変わる。

### 4.10 PDF出力

#### 4.10.1 出力対象の選択
- 時間割グリッド（クラス別）
- 先生別コマ数表

#### 4.10.2 出力方法
- 新規ウィンドウに印刷用HTMLを生成 → `window.print()` → 「PDFとして保存」
- 用紙: A4横

#### 4.10.3 表示内容
- 隔週授業はA/B両方をセル内に表示
- 合同グループは全クラスを1セルにまとめて表示

### 4.11 ファイル保存・読込

#### 4.11.1 保存
- ストア全体をJSON形式でダウンロード
- File System Access API 対応ブラウザ（Chrome/Edge）で元ファイルに直接上書き保存

#### 4.11.2 読込
- JSONファイルを読み込み、ストア全体を復元
- スキーマバリデーション付き（不正なJSONは拒否し、エラーメッセージを表示）

#### 4.11.3 旧版データの互換性
- 新版のスキーマで新規作成。旧版JSONとの直接互換は持たない
- 全機能実装後に、旧版JSON→新版JSONの変換スクリプトを別途作成する

### 4.12 AI支援（Gemini API連携）

#### 4.12.1 機能
- **時間割レビュー**: 現在の時間割を分析し、問題点・改善提案をAIが生成
- **時間割自動生成**: JSON形式で草案を生成 → ストアに適用（あくまで草案。制約の完全充足は保証しない）

#### 4.12.2 設定
- APIキーは `localStorage` に保存
- 使用モデル: ユーザーが設定画面から選択可能

#### 4.12.3 エラーハンドリング
- API接続失敗時: リトライボタン付きのエラーメッセージ表示
- Quota exceeded 時: 別モデルへの切替を提案

### 4.13 ソルバー（ブラウザ内JS）

#### 4.13.1 アルゴリズム
- ランダムグリーディ法（複数回リスタートで最良解を選択）
- Web Worker で実行（UIをブロックしない）

#### 4.13.2 適用モード
- **全上書き**: 既存の配置を全てクリアし、ソルバー結果で置き換え（固定コマは保持）
- **空きコマのみ**: 既に配置済みのコマは保持し、空きコマのみソルバー結果で埋める

#### 4.13.3 制約の扱い
- ハード制約: 必ず満たす。満たせない場合はそのコマを空きのまま残す
- ソフト制約: コスト最小化で可能な限り満たす

#### 4.13.4 UI
- 実行/キャンセルボタン
- 進捗表示（経過時間、試行回数）
- 結果表示（充足率、違反項目の一覧）
- 適用モード選択 → 適用ボタン

### 4.14 複数学年合同授業

- 異なる学年のクラスが同じ時間に合同で授業を行う設定
- 例: 1年全クラスと2年全クラスの「総合的な学習の時間」を同じ時間に設定
- 合同授業の教科・週コマ数・参加学年を設定

## 5. 非機能要件

### 5.1 パフォーマンス
- 想定規模: 3学年 × 最大8クラス + 特支クラス、教員最大80名
- コマ配置操作のレスポンス: 100ms以内
- ソルバー実行: 30秒以内に結果を返す（タイムアウト設定可能）
- バリデーション: コマ配置から200ms以内に結果を更新

### 5.2 データ制限
- 時間割エントリ: 最大 5日 × 6時限 × 30クラス = 900エントリ
- 教員数: 最大100名
- 教科数: 最大30教科

### 5.3 エラーハンドリング方針
- JSON読込失敗: 具体的なエラーメッセージ（どのフィールドが不正か）を表示。部分読込はしない
- ソルバーが解を見つけられない場合: 「制約を満たす配置が見つかりませんでした」+違反した制約の一覧を表示
- API接続失敗: リトライ可能なエラー表示。オフラインでもソルバー以外の全機能は動作

### 5.4 セキュリティ
- Gemini APIキーは `localStorage` に保存。ソースコードにハードコードしない
- JSON読込時のサニタイズ: スキーマバリデーション（Zod等）で不正データを拒否

### 5.5 PWA要件
- Service Worker によるアセットキャッシュ
- オフラインでもアプリ起動・時間割編集が可能（AI支援のみオフラインでは使用不可）
- ホーム画面への追加対応（manifest.json）

## 7. 用語集

| 用語 | 定義 |
|------|------|
| コマ | 曜日×時限×クラスの1つの授業枠 |
| セル | 時間割グリッド上の1つのマス目（1コマに対応） |
| 合同コマ | 複数セルをグループ化し、1つの授業枠として扱う |
| 隔週授業 | A週とB週で異なる教科を交互に実施する授業形態 |
| 抱き合わせ教科 | 2クラス間で同日同時限に特定教科ペアを配置するルール |
| 固定コマ | ソルバーや手動編集で変更不可のコマ（朝礼・学年集会等） |
| ハード制約 | 必ず満たさなければならない制約 |
| ソフト制約 | 可能な限り満たすが、不可能な場合は違反を許容する制約 |
| 教員グループ | 複数教員で1つの教科を分担するグループ |
| 合同クラス | 複数クラスが合同で授業を行う設定 |
| 分割教科 | 合同クラス内で、各クラスに別々の教員を配置する教科 |
