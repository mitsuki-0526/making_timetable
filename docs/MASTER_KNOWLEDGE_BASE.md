# 時間割作成アシストツール：マスターナレッジベース (NotebookLM 用)

このドキュメントは、プロジェクトの全容を NotebookLM にインジェスト（学習）させるためのマスター資料です。仕様、背景、ロードマップを集約しています。

---

## 1. プロジェクトの背景と目的
学校現場における時間割作成は、非常に多くの制約条件（教員の要望、時数、教室、連動ルール等）が絡み合う複雑なパズルです。

- **課題**: 従来のクラウド型AIでは、教員や学校の機密情報を外部に送信することへの懸念があり、導入の障壁となっていた。
- **解決策**: 外部APIを一切使用せず、**完全ローカル環境（スタンドアロン）**で作動する、AIと数理最適化を組み合わせたハイブリッド型ツールを構築する。

## 2. システム・アーキテクチャ（ハイブリッド方式）
PCのスペックを最大限に活用し、ブラウザの制限を超えるために「Pythonバックエンド + Webフロントエンド」の構成を採用。

### 構成要素
1. **フロントエンド (UI)**: React + Vite。直感的な時間割グリッドとドラッグ＆ドロップ機能を提供。
2. **バックエンド (Engine)**: Python (FastAPI)。重い計算処理（AI推論・数理最適化）を担う。
3. **AIモデル**: Gemma 4 (Local)。先生の曖昧な要望を解析し、プログラムが理解できる「制約」に変換。
4. **数理最適化**: Google OR-Tools。数万通りの組み合わせから、制約を最大限満たす時間割を算出。

## 3. 処理フェーズの詳細

### フェーズ1：条件の抽出（AI翻訳）
- **動作**: 先生が「この先生は月曜の1限は会議」「体育の後は連続させないで」といった要望をテキストで入力。
- **AIの処理**: Gemma 4 がこのテキストを読み取り、`{"avoid_consecutive": ["体育"], "unavailable": ["山田", "月", 1]}` といった構造化されたJSONに変換する。
- **工夫**: AIにパズル（配置）自体は解かせず、ルール作りに専念させる。

### フェーズ2：ベースの自動配置（プログラム計算）
- **動作**: AIが作った制約JSONと、標準の設定（規定時数など）を OR-Tools に投入。
- **プログラムの処理**: 1〜3分程度の計算で、80%〜95%程度完成した「ほぼ正解に近い」時間割を自動生成する。

### フェーズ3：手動での微調整（人間による仕上げ）
- **動作**: 自動配置された結果を画面に表示。
- **人間の処理**: エラー（赤枠等）が表示された箇所をドラッグ＆ドロップで入れ替える。
- **UI機能**: コマを掴んで別のマスに重ねると、中身が「入れ替え（Swap）」される仕組み。

## 4. 技術仕様と開発環境

### ソフトウェア要件
- **開発言語**: JavaScript (Frontend), Python 3.x (Backend)
- **GUIフレームワーク**: Electron (Reactアプリをデスクトップアプリとして表示・操作)
- **パッケージング**: PyInstaller / electron-builder を使用し、最終的に **単一の .exe ファイル** として配布。

### 構成ディレクトリ (デスクトップ版)
- `desktop/electron/`: Electron メインプロセス
- `desktop/python/`: Python バックエンド（ソルバー・AI）
- `desktop/src/`: React UI ソース

## 5. 開発ロードマップ

1. **Step 1: UIプロトタイプ制作**: D&D 機能の実装とグリッド表示の最適化。
2. **Step 2: コアエンジンの構築**: Python 側で OR-Tools と Gemma 4 (llama-cpp-python) の連携を確立。
3. **Step 3: 統合と通信**: API を介して UI とエンジンを接続。
4. **Step 4: パッケージ化**: 全依存関係を同梱した配布用 `.exe` の作成。
