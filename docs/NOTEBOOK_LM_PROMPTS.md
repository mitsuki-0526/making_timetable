# NotebookLM 活用プロンプト案

`MASTER_KNOWLEDGE_BASE.md` を NotebookLM に読み込ませた後、以下のような質問をすることで、様々な用途のマニュアルや資料を自動生成できます。

## 1. 先生（ユーザー）向け資料の作成
- 「このツールを初めて使う先生のために、5分で読める『クイックスタートガイド』の下書きを作成してください。」
- 「『フェーズ1：条件設定』でAIにどのように話しかければ良いか、具体的な例文をいくつか挙げてください。」
- 「なぜインターネットに繋がなくてもAIが使えるのか、専門用語を使わずに分かりやすく説明して。」

## 2. 開発・保守者向け資料の作成
- 「新しい技術スタック（FastAPIやOR-Tools）の役割分担を、システム図が書けるような構成案として箇条書きでまとめて。」
- 「Step 2（コアエンジン開発）で具体的に取り組むべきタスクのチェックリストを作成して。」
- 「プロジェクトのディレクトリ構成と、それぞれの役割を詳しく説明して。」

## 3. プロジェクト紹介・プレゼン資料の作成
- 「このプロジェクトの最大の独自性（ユニーク・セリング・ポイント）を3つ、魅力的なキャッチコピーと共に提示して。」
- 「従来のクラウド型AIを使った時間割作成と比較して、このローカル型システムにはどのようなセキュリティ上のメリットがあるか解説して。」
- 「このツールの最終的な目標を、学校現場の負担軽減という観点から要約してください。」

## 4. FAQ（よくある質問）の生成
- 「想定されるユーザー（先生）からの質問を10個予測し、回答案を作成してください。」
- 「スペック要件が厳しい場合に、ユーザーが取るべき対策（モデルの軽量化など）についてドキュメントを生成して。」
