# 時間割作成アシストツール：総合マニュアル

このマニュアルでは、AIと数理最適化を組み合わせたハイブリッド型時間割作成ツールの使い方から、技術的な仕組みまでを解説します。

---

## 1. クイックスタートガイド（5分で読める）

時間割作成の複雑なパズルを、AIとプログラムがサポートします。

### 3つのステップで完成
```mermaid
graph LR
  A[ステップ1: 条件設定] --> B[ステップ2: 自動配置]
  B --> C[ステップ3: 手動微調整]
```

1.  **ステップ1：条件の設定 (AI翻訳)**
    先生方の要望（テキスト）を普段の言葉で入力してください。AIが自動で読み取り、システム用のルールに変換します。
2.  **ステップ2：ベースの自動配置 (計算)**
     「作成開始」ボタンを押すと、プログラムが1〜3分で80%〜95%完成した時間割を自動生成します。
3.  **ステップ3：手動での微調整 (UI)**
    自動配置後の結果に対し、エラー箇所（赤枠など）をドラッグ＆ドロップで入れ替えて（Swap）最終調整を行います。

> [!TIP]
> **AIへの話しかけ方の例**
> - 「山田先生は月曜の1限は会議です」
> - 「体育の後は連続させないで」
> 普段どおりの言葉で伝えるだけで、システムが理解可能な制約として抽出されます。

## 2. 実装済み機能の詳細解説

本ツールには、学校現場の細かなニーズに対応するための高度な機能が備わっています。

### 2.1. 時間割グリッドと編集操作
- **セルの直接集集**: 各マスをクリックして教科と担当教員をプルダウンで選択。
- **動的候補表示**: その時間に空いている先生、かつその教科を担当可能な先生のみが優先して表示されます。
- **複数セル選択 (Ctrl+クリック)**: 複数のマスを同時に青枠で選択し、一括でグループ化設定などが行えます。
- **合同コマの設定 (右クリック)**: 複数クラス合同の授業などを「一つのまとまり」として管理。入れ替え時も連動します。
- **隔週授業 (A週/B週)**: 特定のコマを「A週は国語、B週は書写」のように設定し、PDF出力時も両方を表示できます。

### 2.2. 特別支援学級への自動連動
- **自動マッピング**: 通常学級の時間割を変更すると、設定されたルールに基づき、同学年の「特別支援学級」の該当コマも自動的に更新されます（例：通常[国語] → 特支[自立活動]）。
- **一括管理**: 担任が一人で複数クラスを管理する場合の入力負担を大幅に軽減します。

### 2.3. 教員・教科・マスタ管理
- **教員グループ**: 「道徳」や「総合」など、複数の先生がチームで担当する教科をグループとして一括登録可能。
- **抱き合わせ教科**: Aクラスで「理科」を入れたらBクラスも連動して「理科」をセットする、といった教科間のペアリング設定。
- **配置不可時間の設定**: 出張、会議、通院などの固定された「入れられない時間」を教員ごとに設定。

### 2.4. リアルタイム・バリデーション（自動チェック）
- **持ちコマ数チェック**: 週当たりの規定時数に対して、現在何コマ配置されているかを自動集計。過不足を可視化します。
- **重複・連続授業制限**: 同時限に同じ先生が重なっていないか、また特定の教科が同じ日に集中しすぎていないかを即座に判定。
- **警告表示**: 条件を満たさない箇所は赤枠や色付きの警告で強調され、ミスを未然に防ぎます。

## 3. セキュリティと基本の仕組み

### なぜネット不要でAIが使えるのか？
このツールは、AIの「頭脳」にあたるプログラム（Gemma 4）を、**お使いのパソコンの中に直接組み込んでいます。**

- **外部送信なし**: インターネットにデータを送ることはありません。
- **完全独立**: オフラインで動作するため、教員の個人名や学校の機密情報が外部に漏れるリスクを根本から排除しています。

## 4. 開発・保守者向け技術ガイド

PCスペックを最大限に引き出すため、「Pythonバックエンド + Webフロントエンド」の構成を採用しています。

### システム構成と役割
| コンポーネント | 技術スタック | 主な役割 |
|---|---|---|
| **フロントエンド** | React + Electron | 直感的な時間割グリッドUI、D&D操作 |
| **バックエンド** | Python (FastAPI) | 重い計算処理、AI推論、全体制御 |
| **AI翻訳係** | Gemma 4 (Local) | 自然言語の要望をJSON制約へ変換 |
| **最適化ソルバー** | OR-Tools | 数理モデリングによる最適な配置計算 |

### プロジェクトのディレクトリ構成
- `desktop/electron/`: Electron メインプロセス（基盤）
- `desktop/python/`: コアエンジン（ソルバーとAI）
- `desktop/src/`: React UI（画面レイアウト・操作UI）

## 5. よくある質問 (FAQ)

> [!IMPORTANT]
> **Q: 自動作成で100%完璧な時間割ができますか？**
> A: 本ツールは「80%〜95%程度のほぼ正解に近い状態」を高速に作ることを目指しています。最後の数パーセントのこだわりは、先生の手で直感的に調整していただく設計です。

- **Q: データの漏洩は本当に大丈夫？**
    - A: はい。すべてお手元のPC内で完結するため、クラウドにデータが送られることは一切ありません。
- **Q: なぜブラウザではなくデスクトップアプリなのですか？**
    - A: PCのパワーをフルに使い、複雑なパズルを短時間で解くため、ブラウザの制限を取り払ったデスクトップアプリ形式を採用しています。

## 6. パフォーマンスと対策（スペック不足時）

もし動作が重いと感じる場合は、以下の対策を検討してください。

- **バックグラウンドアプリの停止**: AI推論や最適化計算はCPU/メモリを多く使います。
- **計算時間の調整**: 開発者設定により、ソルバーの計算時間を短くすることで、精度を少し下げつつ動作を軽くすることが可能です。
- **軽量モデルの採用**: 実行環境に合わせて、より軽量なAIモデルへの切り替えも検討してください。
