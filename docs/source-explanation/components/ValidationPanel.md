# 時間割チェック表示

> 対応ソースコード: `src/components/ValidationPanel.jsx`

## このファイルの役割

時間割が完成したかどうかを自動でチェックして、問題がないか教えてくれるコンポーネントです。学校の先生がチェックリストを使って時間割を確認するのと同じです。「この先生、この曜日に授業が多すぎないか？」「この教科、目標の時間数に達しているか？」などを自動でチェックします。

---

## 主な機能

### 教科別コマ数の確認

- **何をするか**: 各クラスの各教科について「目標の時間数」と「現在配置されている時間数」を比較して表示します
  - 例：「1年A組の国語」は目標5コマだけど、現在3コマしか配置されていない場合は赤色で警告
- **表示**:
  - **緑色の枠**：目標に達した
  - **赤色の枠**：目標より少ないか多すぎる
- **ソースコード**: 行209-265（教科別チェック）
- **関連機能**: [store/useTimetableStore.md - getClassSubjectTotals](../store/useTimetableStore.md#getclasssubjecttotals---教科別コマ数を集計する)

### 連続授業日数の警告

- **何をするか**: 同じ教科が何日連続で配置されているかをチェックします
  - 例：「1年A組の体育が月火水と3日連続」→ 上限（例えば2日）を超えているなら警告
- **背景**: 学校によっては「同じ教科は連続5日まで」などのルールがある場合があります
- **アイコン**: ⚠️
- **ソースコード**: 行268-276
- **関連機能**: [store/useTimetableStore.md - getConsecutiveDaysViolations](../store/useTimetableStore.md#getconsecutivedaysviolations---連続日数違反を検出する)

### 固定コマ未反映の警告

- **何をするか**: 「この時間には必ずこの教科」と指定された「固定コマ」が正しく配置されているかチェック
  - 例：「毎週月曜1限は朝礼」と決めている場合、それが入っているか確認
- **警告内容**: 「〇年〇組 月曜1限に『朝礼』が必要ですが『〇〇』が入っています」
- **アイコン**: 🔒
- **ソースコード**: 行10-44（固定コマのチェック関数）、行289-297（表示）

### 先生の労働条件チェック

システムが先生の労働時間をチェックします：

#### 1日最大コマ数超過

- **何をするか**: 先生が1日に教えるコマ数が上限を超えていないかチェック
  - 例：「田中先生は木曜日に6コマ以上教えられない」という上限がある場合、超えていたら警告
- **アイコン**: 👨‍🏫
- **ソースコード**: 行46-60（1日最大コマ数のチェック関数）

#### 連続コマ数超過

- **何をするか**: 先生が1日に連続して教える時間が長すぎないかチェック
  - 例：「田中先生は連続4コマまで」という上限がある場合、5コマ連続で入っていたら警告
- **アイコン**: ⏱️
- **ソースコード**: 行62-86（連続コマ数のチェック関数）

#### 週総コマ数超過

- **何をするか**: 先生が週全体で教えるコマ数が上限を超えていないかチェック
  - 例：「田中先生は週20コマまで」という上限がある場合、超えていたら警告
- **アイコン**: 📅
- **ソースコード**: 行180-192（週総コマ数のチェック関数）

### 教科の配置時限チェック

- **何をするか**: 「この教科は3限と4限にしか配置できない」など、配置時限に制限がある場合、それを守っているかチェック
  - 例：体育が1限に入っていたら「配置可能時限外」として警告
- **アイコン**: 📚
- **ソースコード**: 行88-104（配置時限のチェック関数）

### 午後コマ上限チェック

- **何をするか**: 同じ教科が午後に多すぎないかチェック
  - 例：「体育は1日の午後に2コマまで」という上限がある場合、3コマ午後に入っていたら警告
- **アイコン**: 🌇
- **ソースコード**: 行106-128（午後コマ上限のチェック関数）

### 施設の競合チェック

- **何をするか**: 同じ施設（体育館など）が同じ時間に複数クラスで使われていないかチェック
  - 例：「体育館は月曜1限に1クラスしか使えない」という場合、複数クラスが同時に使っていたら警告
- **アイコン**: 🏫
- **ソースコード**: 行130-154（施設競合のチェック関数）

### 2コマ連続授業の不整合

- **何をするか**: 「この教科は2コマ連続で授業する」と決めている場合、1日のコマ数が偶数になっているかチェック
  - 例：体育が「2コマ連続」と決まっているのに、月曜日に3コマ配置されていたら「奇数コマ」として警告
- **アイコン**: ⏭️
- **ソースコード**: 行156-178（2コマ連続のチェック関数）

---

## チェックの仕組み

1. **リアルタイムチェック**: 時間割を編集するたびに自動でチェックが走ります
2. **複数の検査ルール**: 上記の各チェックが全て同時に実行されます
3. **違反件数のカウント**: 「条件設定の違反 〇件」という風に、何個の問題があるか表示します
4. **色分けされたメッセージ**: 問題の種類ごとに違う色で表示されます

---

## チェック結果の見方

| 色 | 問題の深刻さ |
|---|---|
| 🔴 赤 | 重大な問題（固定コマ未反映、施設の重複使用など） |
| 🟠 オレンジ | 中程度の問題（先生の労働条件超過など） |
| 🟡 黄 | 軽い問題（午後のコマ上限など） |
| 🟣 紫 | 注意が必要（配置時限外、2コマ連続の不整合など） |

---

## 実装のポイント

- **メモ化**: 同じデータで何度もチェックする場合、結果をキャッシュして処理を高速化しています（`useMemo` を使用）
- **条件ベース**: チェックルールは「設定」で管理されている条件に基づいています
  - 条件を変更すれば、チェック内容も自動で変わります
