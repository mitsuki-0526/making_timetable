# 条件設定画面

> 対応ソースコード: `src/components/ConstraintsModal.jsx`

## このファイルの役割

このファイルは、時間割を作るときに「これはしてはいけない」「これはこうあるべき」という細かい制約・ルールを登録する場所です。

[マスタ設定画面](./SettingsModal.md)が「学校の基本情報（教科、先生、クラス）」を決めるのに対して、この条件設定は「時間割作成時の細かい制約」を決めます。

例えば「体育館の授業は1日1つのクラスしか使えない」「○○先生は月曜日の5限は出張で配置不可」といった制約です。

---

## 主な機能

このモーダル（ポップアップ窓）には **6つのタブ**があります。

### タブ①：固定コマ（🔒）

- **何をするか**: 「月曜の朝礼は全校で朝会」「毎週金曜5限は学年会」など、**毎週決まっている授業**を「固定コマ」として登録します
  - これらは時間割ソルバーによって動かされません

- **具体的な操作**:
  1. **適用範囲を選択**:
     - **全校共通**: すべてのクラスが同じ時間に同じ授業（例：全校朝会）
     - **学年指定**: 特定の学年のすべてのクラスが同じ授業（例：3年生全員で学年会）
     - **クラス指定**: 特定のクラスだけの固定授業（例：1年A組の特別な活動）
  
  2. **曜日・時限を選択**: 月曜1限など、固定する日時を選ぶ
  
  3. **教科を選択**: その時間に何をするかを選ぶ
  
  4. **ラベル（任意）**: 「全校朝会」など、わかりやすい説明を入力
     - 後で「どの固定コマだか」を見て判断しやすくなります
  
  5. **追加ボタンを押す**: 固定コマが登録される

- **表示**: 登録された固定コマは表の下に一覧表示されます

- **ソースコード**: 行9-155
- **関連ファイル**: [useTimetableStore.md#addfixedslot](../store/useTimetableStore.md#addfixedslot---固定コマを追加する)

---

### タブ②：時間帯（⏰）

- **何をするか**: **昼休みの位置**を設定します
  - これによって「午前」「午後」の定義が決まり、後で「午後の授業は1日1コマまで」といった制約が使えるようになります

- **具体的な操作**:
  1. **「🍱 昼休みは」から時間を選択**:
     - 例：「4限と5限の間」を選ぶと
     - 午前 = 1限～4限
     - 午後 = 5限～6限
  
  2. **設定が確定**すると、下に「☀️ 午前」「🌇 午後」の時限が視覚的に表示されます

- **意義**:
  - 教科ごとに「午後は最大1コマまで」といった制約を設定するときに、この昼休みの位置が基準になります
  - 例：「体育は午前に集中させたい」という場合、午前の定義が必要です

- **教科配置タブとの連携**:
  - 「📚 教科配置」タブで「午後1日上限」を設定するときに、ここで定めた「午後」の時限が対象になります

- **ソースコード**: 行160-241
- **関連ファイル**: [useTimetableStore.md#updatelunchperiod](../store/useTimetableStore.md#updatelunchperiod---昼休みの時限を設定する)

### タブ③：教員制約（👨‍🏫）

- **何をするか**: 各先生ごとに、授業コマ数の上限・担任クラス・空きコマの集約を設定します
  - 例：「田中先生は週に最大18コマまで」「山田先生は1年A組の担任」といった制約

- **具体的な操作** (各先生の行で):
  1. **1日最大**: その先生が1日に何時間まで授業を持てるか（1～6時間）
     - 例：1日に4限以上授業を持たせたくない場合は「4」と入力
  
  2. **連続最大**: 休み時間なしで授業できる最大時間数（1～6時間）
     - 例：「2」と入力すれば、最大2時限連続まで
  
  3. **週最大**: 1週間に何時間まで授業を持てるか（1～30時間）
     - 例：「18」と入力すれば、週18時間まで
  
  4. **担任学年**: その先生が担任を持つ学年（1～3年、なしでもOK）
     - ドロップダウンから選択
  
  5. **担任クラス**: 担任学年を選んだ後、具体的なクラスを選択
     - 例：「1年」を選んでから「A組」を選択
  
  6. **空きコマ集約**: チェックボックス
     - ON にすると「授業の合間に空き時間を作らないよう最適化」します
     - 例：1限、2限授業、4限授業があるとき、3限も授業を入れて「空き時間を詰める」

- **空欄の意味**: 空欄のままなら制限なし

- **ソースコード**: 行246-337
- **関連ファイル**: [useTimetableStore.md#updateteacherconstraintsettings](../store/useTimetableStore.md#updateteacherconstraintsettings---教員の制約を設定する)

### タブ④：施設制約（🏫）

- **何をするか**: 学校にある「同時に1クラスしか使えない施設」（体育館、理科室、図書館など）を登録し、教科と結びつけます
  - ソルバーが「同じ時限に同じ施設を複数クラスが使わない」ように自動制御します

- **具体的な操作**:
  1. **施設名を入力**: 「体育館」「理科室」など
  2. **追加ボタンを押す**: 施設が登録される
  3. **教科と施設の紐付け**:
     - 登録した施設ごとに「どの教科がこの施設を使うか」をドロップダウンで選択
     - 例：「体育館」→「体育」を選ぶ
     - 「体育」の授業が配置されるたびに、その時間帯は他のクラスは体育館を使わないようになります

- **実践例**:
  - 「体育館」を登録 → 「体育」「ダンス」などの教科を紐付け
  - すると、1年A組が月1限に体育館で体育をしているとき、同じ月1限に他のクラスが体育館で授業することは避けられます

- **ソースコード**: 行342-425
- **関連ファイル**: 
  - [useTimetableStore.md#addfacility](../store/useTimetableStore.md#addfacility---施設を追加する)
  - [useTimetableStore.md#removefacility](../store/useTimetableStore.md#removefacility---施設を削除する)
  - [useTimetableStore.md#updatesubjectfacility](../store/useTimetableStore.md#updatesubjectfacility---教科と施設を紐づける)

### タブ⑤：教科配置（📚）

- **何をするか**: 各教科ごとに「どの時限に配置できるか」「1日何コマまでか」「午後に集中させない」など、配置のルールを細かく設定します

- **具体的な操作** (各教科の行で):
  1. **配置可能時限** (ボタン群):
     - 🔵 **青ボタン** = 午前時限
     - 🟡 **黄ボタン** = 午後時限
     - クリックすると、その時限への配置が ON/OFF になります
     - **例**：「国語は午前の1,2,3,4限のみに配置」→ これらのボタンを青くしておく
     - ボタンが何も押されていない（全て OFF）なら「制限なし」
  
  2. **1日最大コマ**: 1日に最大何コマまで配置するか
     - 例：「2」と入力すれば、1日に「国語」は最大2コマまで
  
  3. **午後1日上限**: 午後に何コマまで配置するか
     - 例：「1」と入力すれば、午後は最大1コマ（つまり、同じ日の午前と午後には同じ教科を両方配置しない）
  
  4. **午後分散**: チェックボックス
     - ON にすると、午後の授業を「異なる曜日」に分散させます
     - 例：午後の体育が週3コマなら、月・水・金の3日に分けて配置
  
  5. **全体分散**: チェックボックス
     - ON にすると、1週間を通してなるべく均等に分散
     - 例：週4コマの国語なら、月火水木に1コマずつ配置
  
  6. **2コマ連続**: チェックボックス
     - ON にすると「2時限連続で配置」が基本ルール
     - 理科実験や美術など、2時限必要な教科に使用

- **昼休みの境界**: 画面に「現在: 4限まで午前 / 5限以降午後」など表示されます
  - これは「⏰ 時間帯」タブで設定した値です

- **ソースコード**: 行430-553
- **関連ファイル**: [useTimetableStore.md#updatesubjectplacement](../store/useTimetableStore.md#updatesubjectplacement---教科の配置ルールを設定する)

### タブ⑥：隔週授業（🔄）

- **何をするか**: 「A週は音楽、B週は図工」のように、同じ時間帯で週ごとに異なる教科を交互に行う授業を設定します
  - **必須**: [SettingsModal](./SettingsModal.md) の「教科・連動ルール」タブで、該当教科の「規定時数」をあらかじめ設定しておく必要があります
    - 例：「1年通常の音楽=2、図工=2」と設定しておいて、隔週ペアでcount=2を設定すれば、同じコマが「A週:音楽(2コマ) / B週:図工(2コマ)」になります

- **具体的な操作**:
  1. **クラス区分を選択**: 「1年通常クラス」「1年特支クラス」など
     - 選択すると、そのクラスの規定時数に含まれる教科のみが選べるようになります
  
  2. **A週の教科**: メインの教科を選択（例：音楽）
  
  3. **B週の教科**: 交互に行う教科を選択（例：図工）
     - A週と異なる教科を選ぶ必要があります
  
  4. **隔週スロット数**: A週と B週のそれぞれに配置される同一コマの数
     - 例：「2」を入力すれば、2つの同じ時限が「A週:音楽 / B週:図工」になります
  
  5. **追加ボタンを押す**: 隔週授業ペアが登録されます

- **ソルバーの動き**:
  - ソルバーが実行されるとき、A週教科のスロットには自動的に B週教科が「alt_subject（代替教科）」というタグで付けられます

- **重要**:
  - required_hours の設定と「隔週スロット数」が合致していないと、ソルバーが期待通りに動作しません
  - 例：音楽=2、図工=2、隔週count=2 が正しい組み合わせ

- **ソースコード**: 行558-682
- **関連ファイル**:

  - [useTimetableStore.md#updatealtweekpair](../store/useTimetableStore.md#updatealtweekpair---隔週授業ペアを更新する)

## データの流れ

1. ユーザーが「条件設定」ボタンを押す → ConstraintsModal が表示される
2. 各タブで固定コマ・時間帯・教員制約・施設・教科配置・隔週授業を登録・編集する
3. 設定データは [useTimetableStore](../store/useTimetableStore.md) に保存される
4. **ソルバー実行時**に、これらの制約がすべて時間割生成に反映される
   - 「制約を満たす時間割を生成する」という最適化問題が解かれます
