# ローカルAI連携（Ollama）

> 対応ソースコード: `src/lib/localLLM.js`

## このファイルの役割

このファイルは、自分のパソコン上で動く「ローカルAI」と通信するための道具です。

**Gemini（クラウドAI）との違い**：
- **Gemini**: インターネット経由でGoogle のサーバーのAIを使う（高性能だが、ネット接続が必要）
- **ローカルAI**: 自分のパソコン内で動くAI（ネット接続不要だが、パソコンが遅い場合は時間がかかる）

**Ollama（オラマ）とは**：
ローカルAIを簡単に使うためのソフトウェア。パソコンに Ollama をインストール → モデル（AI）をダウンロード → このアプリから利用、という流れです。

---

## 主な機能

### callLocalLLM --- ローカルAIに問い合わせる

- **何をするか**: 質問をローカルAI（Ollama）に送信して、回答を受け取ります。
- **ソースコード**: `src/lib/localLLM.js` 行39-78
- **パラメータ**: `prompt`: AIに送る質問文
- **返す値**: AIからの回答テキスト
- **処理フロー**:
  1. 保存されているOllama接続先URL（デフォルト: `http://localhost:11434`）を取得
  2. 保存されているモデル名（デフォルト: `gemma3`）を取得
  3. `/api/chat` エンドポイントにリクエストを送信
  4. AIからの回答を取得して返す

- **エラー処理**: 
  - Ollamaが起動していない → 「Ollamaに接続できませんでした」
  - ネットワークエラー → エラーメッセージ
  - モデルの応答エラー → 詳しいエラー内容

- **パフォーマンス設定**:
  - 温度（temperature）: 0.4（やや控えめ、回答がブレにくい）
  - 最大出力トークン: 8192個

- **特徴**: インターネット接続が不要。学校のセキュリティが厳しく、外部APIが使えない場合に活躍します。

### testOllamaConnection --- 接続をテストする

- **何をするか**: Ollamaが起動しているか、接続できるか確認します。Ollamaのバージョン情報を取得することで疎通を確認。
- **ソースコード**: `src/lib/localLLM.js` 行81-94
- **返す値**: `"Ollama v0.X.X に接続しました。"` のようなメッセージ
- **エラー処理**: 接続できない場合は、Ollamaが起動しているか確認するよう促すメッセージ
- **使用場面**: ユーザーが【マスタ設定】で「接続テスト」を押したときに実行
- **関連ファイル**: [マスタ設定画面](../components/SettingsModal.md#AI設定タブ) — ここから呼ばれる

### checkModelAvailable --- モデルの存在を確認する

- **何をするか**: Ollama上に、選択したモデルがインストールされているか確認します。
- **ソースコード**: `src/lib/localLLM.js` 行97-119
- **処理フロー**:
  1. Ollamaに「利用可能なモデル一覧」をリクエスト
  2. 設定されているモデルが含まれているか確認
  3. 含まれていなければ、ダウンロード方法をエラーメッセージに含める

- **返す値**: 成功時 → `"モデル「gemma3」が利用可能です。"`
- **エラー処理**: 
  - モデルが見つからない → `"モデル「gemma3」が見つかりません。\nollama pull gemma3 を実行してダウンロードしてください。\n（インストール済みモデル: llama3.2, ...）"`
  - 他のエラー → Ollama接続エラーなど

- **重要な役割**: ユーザーが「このモデルを使いたい」と選択したけれど、まだダウンロード（インストール）されていない場合に、「このコマンドを実行してダウンロードしてね」と教えてくれます。

## 接続先・モデル設定の保存・取得

### getStoredOllamaUrl / setStoredOllamaUrl --- 接続先の保存・取得

- **何をするか**: Ollama のアクセスURLをブラウザのローカルストレージに保存・取得します。
- **ソースコード**: 
  - `getStoredOllamaUrl`: `src/lib/localLLM.js` 行20
  - `setStoredOllamaUrl`: `src/lib/localLLM.js` 行21-27
- **デフォルト値**: `http://localhost:11434`
  - `localhost`：同じパソコン
  - `11434`：Ollama のデフォルトポート番号
- **変更が必要な場合**:
  - Ollama を別のポート番号で起動した場合 → URL を変更
  - 別のパソコンの Ollama を使う場合 → そのパソコンのIPアドレスに変更
    - 例：`http://192.168.1.100:11434`（同じネットワーク内の別パソコン）
- **セキュリティ注意**: Ollama のセキュリティ設定に注意してください。パスワード保護がない場合、ネットワーク内の他のパソコンから不正にアクセスされる可能性があります。

### getStoredModel / setStoredModel --- モデル設定の保存・取得

- **何をするか**: どのAIモデル（Gemma, Llama など）を使うか、ブラウザに保存・取得します。
- **ソースコード**: 
  - `getStoredModel`: `src/lib/localLLM.js` 行29
  - `setStoredModel`: `src/lib/localLLM.js` 行30-36
- **利用可能なモデル**:
  | モデル名 | 推奨度 | 説明 | メモリ必要量 |
  |---------|------|------|-----------|
  | gemma3 | ⭐推奨 | バランス型、実用的 | 約10GB |
  | gemma3:4b | | 軽量、低スペック向け | 約2GB |
  | gemma3:12b | | 高性能 | 約16GB |
  | gemma3:27b | | 最高性能 | 約50GB |
  | llama3.2 | | Llamaシリーズ | 約4GB |

- **インストール方法**:
  ```
  ollama pull gemma3        # デフォルトバージョンをダウンロード
  ollama pull gemma3:4b     # 軽量版をダウンロード
  ollama pull llama3.2      # 別のモデルをダウンロード
  ```

- **関連ファイル**: [マスタ設定画面](../components/SettingsModal.md#AI設定タブ) — ここでモデルを選択

## 定数・設定値

### DEFAULT_OLLAMA_URL
- **値**: `'http://localhost:11434'`
- **意味**: デフォルトの接続先。ローカルパソコン上の Ollama

### DEFAULT_MODEL
- **値**: `'gemma3'`
- **意味**: デフォルトで使用するAIモデル

### AVAILABLE_MODELS
- **値**: モデル選択肢の配列
- **含まれる情報**: id（モデル名）、name（日本語表示名）、recommended（推奨フラグ）

## Ollama のセットアップ（参考情報）

### インストール手順

1. **Ollama のダウンロード**
   - https://ollama.ai にアクセス
   - お使いのOS（Windows/Mac/Linux）用をダウンロード

2. **インストール**
   - ダウンロードしたファイルを実行
   - 通常の「次へ」「次へ」でOK

3. **モデルのダウンロード**
   - コマンドプロンプト（Windows）またはターミナル（Mac/Linux）を開く
   - 以下を実行：
     ```
     ollama pull gemma3
     ```
   - 初回は1〜2分待ちます（モデルサイズにより異なる）

4. **起動確認**
   - Ollamaアプリケーションを開く（通常は自動起動）
   - `http://localhost:11434` でアクセス可能か確認

### 使用開始

- このアプリを開く
- 【マスタ設定】→【AI設定】タブ
- 「ローカルAI（Ollama）」を選択
- 「接続テスト」ボタンで動作確認

## Gemini との使い分けガイド

| 項目 | ローカルAI（Ollama） | Gemini（クラウドAI） |
|------|------------|----------|
| **ネット接続** | 不要 | 必須 |
| **速度** | 遅い（数十秒） | 速い（数秒） |
| **精度** | 中程度 | 高い |
| **セキュリティ** | データがパソコン内に留まる | Google に送信される |
| **セットアップ** | 複雑（Ollamaインストール必須） | 簡単（APIキー入力のみ） |
| **推奨用途** | セキュリティ重視、ネット不安定 | 精度重視、セットアップ簡単 |
| **費用** | 無料（パソコン性能に依存） | 無料〜有料（使用量による） |

## 通信の流れ（図解）

```
1. ユーザーがアプリで「AIレビュー」をクリック
   ↓
2. buildReviewPrompt() が詳しい質問文を作成
   ↓
3. callLocalLLM(question) が Ollama に送信
   ↓
4. 自分のパソコン内で AI が処理（数十秒かかる）
   ↓
5. AI の回答がブラウザに返ってくる
   ↓
6. 回答を画面に表示
```

**特徴**: 
- 全ての処理が自分のパソコン内で完結
- インターネット接続が不要
- データが外部に漏れない
- ただし処理が遅い（パソコンのスペック次第）

## トラブルシューティング

### 「Ollamaに接続できませんでした」というエラーが出る

**原因と対策**:
1. Ollama が起動していない
   → Ollama アプリを起動してください

2. ポート番号が違う
   → Ollama を別のポートで起動した場合は、設定を変更してください
   → 【マスタ設定】→【AI設定】で接続URL を編集

3. ファイアウォールでブロックされている
   → Windows ファイアウォール設定でOllamaを許可してください

### 「モデルが見つかりません」というエラーが出る

**対策**:
エラーメッセージに表示されているコマンドを実行してください：
```
ollama pull gemma3
```

### AI の回答が遅い

**原因と対策**:
- ローカルAIは遅いもの（数十秒かかるのが正常）
- パソコンのスペックが低い場合はさらに遅い
- 軽量版モデル（gemma3:4b など）を試す
- Gemini の方が速いので、ネット環境がいい場合はそちらを使う

### Ollama が起動しない

**対策**:
- アプリを再インストール
- 公式ドキュメント https://ollama.ai を参照
