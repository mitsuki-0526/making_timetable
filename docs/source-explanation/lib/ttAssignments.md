# TT設定ヘルパー

このファイルは、TT授業の設定と実際の時間割データをつなぐための共通処理です。

## 何をしているか

`src/lib/ttAssignments.ts` は、

- どの授業が TT 設定に一致するかを探す
- 一致した授業に、参加する先生全員の一覧をまとめて付ける
- 画面操作と自動生成 solver が同じ判定ルールを使えるようにする

という役割を持っています。

## なぜ必要か

TT設定は「この学年群の中のどのクラスで、どの教科群では、複数の先生が一緒に入るか」というルールです。

この判定を画面側と solver 側で別々に書くと、

- 手で教科を入れた時だけ TT になる
- 自動生成では別の先生が選ばれる
- 保存データや表示の先生一覧がずれる

といった食い違いが起きやすくなります。

そのため、このファイルで TT 設定の探し方を一つにまとめています。

## 主な処理

- `getTtAssignmentSubjects`
  1つの TT 設定に複数教科が入っている場合でも、重複を除いた教科一覧にそろえます。旧データの単一 `subject` もここで吸収します。

- `getTtAssignmentGrades`
  1つの TT 設定に複数学年が入っている場合でも、使いやすい学年一覧にそろえます。旧データの単一 `grade` もここで吸収します。

- `getTtAssignmentTargetClasses`
  指定した学年に対して、その学年で有効な対象クラス一覧を返します。新しい `target_classes` を優先しつつ、旧データの `class_names` も後方互換で読めます。

- `getTtAssignmentTargetClassMap`
  学年ごとの対象クラス表をまとめて取り出します。設定画面の一覧表示や左パレットの要約表示は、この結果を使って「1年:1組・2組 / 2年:3組」のように見せます。

- `findMatchingTtAssignment`
  学年・クラス・教科が一致する TT 設定を探します。1つの TT 設定に複数学年や複数教科を入れている場合も、この一覧から一致判定します。対象クラスは「学年ごと」に別々に見ます。

- `buildTtAssignmentTeacherSnapshot`
  一致した設定から、代表の先生と参加先生一覧を作ります。

- `haveSameTeacherSet`
  合同授業のように複数クラスが同時に同じ授業をするとき、参加先生の組み合わせが同じかを確認します。

## 他ファイルとの関係

- `src/store/slices/timetableSlice.ts`
  手で教科を入れた時に、TT設定の先生を自動反映します。さらに、パレットから TT 設定をドラッグした時にも同じ判定を使います。

- `src/lib/teacherAssignment.ts`
  抱き合わせや連動配置で教科をコピーした時にも、同じ TT 設定を使います。

- `src/lib/jsSolver.worker.ts`
  自動生成の中でも、この helper を使って TT 設定どおりの先生チームを選びます。
