# exe 配布方針比較表

## インストーラー形式 vs 単体 exe

以下は Windows デスクトップ版を配布する際の 2 つの方針比較です。

---

## 1. NSIS インストーラー形式（推奨）

### 概要
デスクトップアプリケーションの標準形式。ユーザーがダウンロードした `.exe` ファイルを実行すると、インストールウィザードが起動し、アプリケーションをWindows システムにインストールします。

### メリット
- **ユーザー体験が良い**
  - 「次へ」「インストール」のウィザード UI で直感的
  - アンインストール機能が標準で付属
  - スタートメニュー / デスクトップショートカット自動作成
- **システム統合**
  - `Program Files` 配下の標準ディレクトリにインストール
  - Windows の「アプリと機能」から管理・アンインストール可能
  - レジストリに登録され、バージョン管理が容易
- **バージョン管理**
  - 更新時に古いバージョンを自動削除
  - バージョン番号が明示的に表示される
- **配布実績**
  - 一般的な Windows アプリケーション（Word、Slack など）と同じ形式
  - ユーザーが慣れている方式

### デメリット
- ダウンロードサイズがやや大きい（数 MB）
- インストール処理に数秒～数十秒必要
- インストール先をユーザーが選択可能（サポート複雑化のリスク）

### 推奨用途
- **学校 IT 部門による集中インストール**
- **複数台への一括導入**
- **定期的な更新配布**

---

## 2. 単体 exe 形式

### 概要
インストール不要。ダウンロードした `.exe` ファイルをそのまま実行。一時ファイルは Windows の Temp フォルダに自動作成されます。

### メリット
- **超簡単な配布**
  - ダウンロード 1 クリック → 即実行
  - インストール処理なし
  - ファイルサイズが若干小さい
- **ユーザー依存性が低い**
  - 管理者権限不要（ほとんどの場合）
  - システム設定変更なし
  - USB メモリから実行可能

### デメリット
- **ユーザー体験が低い**
  - 毎回フルサイズのファイルをダウンロード必要
  - 起動に数秒～十数秒掛かる
  - アンインストール機能がない（手動削除）
- **管理性が低い**
  - バージョン管理が曖昧
  - 複数バージョンが混在しやすい
  - スタートメニュー登録なし
- **学校 IT 部門での管理が困難**
  - インストール履歴がシステムに記録されない
  - どのマシンにインストールされているか追跡困難

### 推奨用途
- **1 回限りの試用**
- **個人ユーザーの一時的な使用**
- **ネットワーク接続が不安定な環境**

---

## 推奨方針: NSIS インストーラー形式

### 理由

1. **学校環境への適合性**
   - 学校の IT 部門は Windows アプリの標準形式に慣れている
   - グループポリシー / WSUS などの配布システムと相性が良い
   - バージョン管理が明確なため、管理者の負担が少ない

2. **ユーザー体験**
   - 一度インストールすれば、以後は起動が高速
   - アンインストール機能により、削除が簡単

3. **将来の拡張に対応**
   - .NET Framework や他の依存ライブラリが必要になった場合、容易に含める
   - 自動更新機能を後から追加可能

## Tauri での実装

`making_timetable` プロジェクトでは、既に **NSIS インストーラー形式** で設定されています：

```json
// src-tauri/tauri.conf.json
"bundle": {
  "active": true,
  "targets": ["nsis"],
  "icon": [...]
}
```

### ビルドコマンド

```bash
pnpm run tauri:build
```

このコマンド実行後、以下に NSIS インストーラーが生成されます：

```
src-tauri/target/release/bundle/nsis/
└── 時間割作成ツール_1.0.0.exe
```

---

## 単体 exe への変更方法（参考）

もし将来、単体 exe への変更が必要な場合：

```json
// src-tauri/tauri.conf.json
"bundle": {
  "active": true,
  "targets": ["nsis", "portable"],  // "portable" を追加
  "icon": [...]
}
```

`pnpm run tauri:build` 実行時、NSIS インストーラーと単体 exe の両方が生成されます。

```
src-tauri/target/release/bundle/
├── nsis/
│   └── 時間割作成ツール_1.0.0.exe      ← インストーラー
└── portable/
    └── 時間割作成ツール.exe             ← 単体実行版
```

## まとめ

| 項目 | NSIS インストーラー | 単体 exe |
|------|------------------|---------|
| セットアップ難度 | ⭐⭐⭐ 簡単 | ⭐ 超簡単 |
| 学校 IT 部門での管理 | ⭐⭐⭐ 良好 | ⭐ 困難 |
| ユーザー体験（起動速度） | ⭐⭐⭐ 高速 | ⭐ 遅い |
| アンインストール | ⭐⭐⭐ 標準機能 | ⭐ 手動削除 |
| **推奨度** | **⭐⭐⭐ 推奨** | ⭐ 非推奨 |

**→ NSIS インストーラー形式でのデプロイを推奨します。**
